Picture Puzzle Probably Programmed Poorly Part Pone

Damnit. Almost had a perfect aliteration. Ah well.

So… It’s done. The first puzzle of the website is up. There I was in all my ignorance thinking that a sliding puzzle would be an easy little starter! No problem! They’re not THAT complicated.

Well, they fucking are. At least programatically. The first thing that needed to be done was to slice the ASCII art into 9 little squares. This was actually easier than I thought it was going to be. I feed it the how many characters height and width are for each tile and it reads the contents of that square and saves them to an array starting from top left and working right. I then assign an index in that array to be the “empty” tile. In my case it was the fourth tile [array position 3].

    Before we can do anything, however, we need to set up our board and the solve state.

var board = [0, 1, 2, 3, 4, 5, 6, 7, 8];
var emptyIndex = 3;
const solved = [0, 1, 2, 3, 4, 5, 6, 7, 8];

    And… that’s it. I know, it’s pretty whelming. The board variable will store the current state of our game, while the solved is what we’re checking board against. After each move the player makes, we’ll compare the current board and solved and if they match, you win! If not, the game continues. It’s actually pretty important that the board is in numerical order for reasons I hope will become clear as we go.

    One thing to note before we get into the thick of things. While the computer just sees an array as… well, an array. A set of numbers all lined up, we need to think of the array in terms of a 3x3 grid. So to US the bored array “looks” like this. Oh, and emptyIndex can be whatever the hell you want it to be. I just chose 3 because in my art the 4th tile was just an outer edge with nothing else happening, so losing that visual cue didn’t hurt anything.

0 1 2
3 4 5
6 7 8

    The shuffle was a little trickier. If you just randomly order the array then about half the time you’ll actually come up with a scramble that is literally impossible to solve as there are some combinations that cannot be made with legal moves. So, how do we solve this? Here’s the code below. Take a moment to see if you can follow the logic before reading on. I’ve always found that puzzling (heh) things out myself helps me remember better. Remember, our goal here is to translate a line of numbers [0,1,2,3,4,5,6,7,8] into a 3x3 grid.



const maxShuffle = 300;
var currentShuffle = 0;
const candidates = [];
const left = emptyIndex - 1;
const right = emptyIndex + 1;
const above = emptyIndex - COLS;
const below = emptyIndex + COLS;

if (left >= 0 && Math.floor(left / COLS) === Math.floor(emptyIndex / COLS)) {
candidates.push(left);
}

if (right < ROWS * COLS && Math.floor(right / COLS) === Math.floor(emptyIndex / COLS)) {
candidates.push(right);
}

if (above >= 0) {
candidates.push(above);
}

if (below < ROWS * COLS) {
candidates.push(below);
}

    Here’s the rundown of the main parts we need to keep track of.

    emptyIndex is the “hole” in our puzzle. That is just set to 3 during an initialization function that sets up some variables, calls Shuffle(), renderBoard() and some other stuff we’ll get to later.

    COLS is how many columns we have. Again, it’s just a constant set to 3. Ditto for ROWS, giving us a 3x3 grid.

    Finally, ROWS * COLS = 9. We need to know how many tiles there are in our puzzle total (remember, arrays start at 0 so the ninth tile of our puzzle is only number 8 in the array. The first tile is number 0.)

    Got it so far? Good.

    Now we need to find neighbors and verify them. The tile to the LEFT of our hole is emptyIndex - 1. However, we need to know if we’ve accidentally wrapped around to the row above. In our case, we have 3-1=2 and array index 2 is the rightmost tile of the top row. That’s where the math.floor comes in. math.floor just says “do the equation, and then give me the result rounded DOWN to the nearest integer”. So any number from 1-1.999999~ would just be rounded down to 1. Pretty handy if we don’t want a pesky decimal getting in the way. For our check right now, math.floor(left / COLS) returns 0.66, but when floored we round down to 0. So, our current check looks like this if 2 >= 0 AND (2/3=0) make sense? All left >= 0 is doing is making sure we haven’t gone outside the grid. -1 would be to the left of 0 and that’s no good. Now, here’s the second part. === means compare the left side to the right side without changing variable types (I won’t get into it here but using just == can result in some really fucky results so use === whenever you’re comparing two values unless you really, REALLY know what you’re doing and have a very good reason). On the right side of our comparitor we have math.floor (emptyIndex / COLS) which would resolve like so 3/3=1.

    Woops. Last time I checked, 0 does NOT equal 1. This is how we know we’ve accidentally wrapped around to the next row. So we now know that the array index that’s to our [3] left [2] is in a different row. We ignore that number and move on.

    To check the number to our right, we use the same sort of logic. But rather than seeing if the number to our right is greater than 0 (we already know it will be anyway) we need to see if we’ve gone too high. So if right ever returns a 9 or above, we know we can ignore that. Then we just need to do the same check we did for the left neighbor. 4/3=1.3~ which, when floored, returns 1. Now our check looks like if 4 < 9 AND (4/3=1) which are both true so now we can compare it to our emptyIndex location just like we did before. if 4 < 9 AND (4/3=1) === 1 Now we’re getting somewhere. 1 is definitely equal to 1 and 4 is less than 9 so we’ve found our first valid neighbor! We shove that wonderful little 4 into our candidates array and move on to the next checks.

    By now you should have an idea of the way this works. To check the tile above, we don’t need to worry whether or not the number we’re checking is in a different column so all we need to do is check what row it is in. The above check is easy. We’re just looking to see if we’ve gone outside the borders by hitting a negative number (which would be in a row “above” our top). And that’s all we have to do for the above check. Any number we review will always be valid as it’ll only ever return either the top row (0), the middle row(1) or “above” our grid (-1). In our case, from tile 3, 0 is definitely equal or greater than 0 so we know that the 0 tile is a neighbor. We now have [0,4] in our array. One more check to go.

    Below is a little trickier, but not by much. We already used similar logic in our check for the right hand neighbor. All we do here is see if the number return is less than 9. 3+3=6, which is less than 9, so we save that one as well.

    I hope I explained that clearly enough. If not, just pick a random number between 0-8 and run it through the equations and it should start to make sense. I actually had to draw out the grid on paper and scribble a bunch of numbers and fucking arrows everywhere to make sense of it and figure out how the hell to calculate it all.

    Anyway, now that we’ve gotten our three neighbor candidates [0,4,6] we can move on to actually shuffling them around!

    Here’s the next bit of code. In the live version, lastSwap is set at the very beginning of the shuffle() function but it didn’t become relavent until now. Once again, see if you can figure it out. This part is a LOT easier to understand.

let lastSwap = -1;
const filtered = candidates.filter(c => c !== lastSwap);
const chosen = filtered[Math.floor(Math.random() * filtered.length)];
board[emptyIndex] = board[chosen];
board[chosen] = 3;
lastSwap = emptyIndex;
emptyIndex = chosen;

    The first thing of note is lastSwapped. Notice how it’s set to -1 and not "" which would give us an empty variable. This is a requirement for the filtered constant.

    Now, here’s where things get stupid and is one of the reasons I fucking hate javascript sometimes. To a novice, if you were to look at candidates.filter(c => c !== lastSwap) what would be your first intuition? Probably that I’m checking if a candidate number is equal to or greater than itself, yeah? Except that’s not what it is at all. (That would actually be >= and yes, the order FUCKING MATTERS A LOT) instead, that’s called an arrow function and if you’re not careful they’ll trip you up. That entire line could be written out as

const filtered = [];
for (const c of candidates) {
  if (c !== lastSwap) {
    filtered.push(c);
  }
}

      But that’s too much writing and is just more chances for a bug to slip in because we fat fingered something or forgot a semicolon or whatever else. Think of the .filter as shorthand for the above.

  So, let’s break down the call. candidates is obviously our array that we built previously. .filteris the new method here. Filter will automatically loop through each index in an array for us. The first c is just whatever variable we want. It won’t matter outside of that line, so in programming it’s common practice to give them single letter names. Once we move to the next line of the code, we will never look at c again. =>is passing that variable into our loop. The real thing happening is c !== lastSwap. Here we are comparing each of our indexes in candidates to make sure that we don’t choose the same two tiles to swap twice in a row. After we choose a tile to swap with the empty space, we set lastSwapped to the old empty index. On the next shuffle step, filter will remove that index from candidates. That prevents the algorithm from just bouncing the same tile back and forth.

    Now that we’ve made sure that none of our candidates have just traded places with our emptyIndex, we can move on.

const chosen = filtered[Math.floor(Math.random() * filtered.length)];
board[emptyIndex] = board[chosen];
board[chosen] = 3;
lastSwap = emptyIndex;
emptyIndex = chosen;

    That first line is easy. We need to choose a tile to swap with. (Math.random() * filtered.length) is what’s doing the actual selection. Random( ) chooses a number between 0 and UP TO but not including 1. It’s a ridiculously long decimal. Something like 15 places so it could come up with 0.999999999999999 but never hit 1. That’s a problem for us, because we need to compare whole numbers and floor would just return 0 every time. The fix is to multiply our random() result by the length of our array. Since we haven’t made any swaps yet, our filtered array is currently indentical to our candidates array. [0,4,6]. So when we multiply by 3, we will recieve any number between 0 and >3. Once that is floored, it’ll return either a 0, 1 or 2. (Once again, arrays start at 0. This will be the last time I mention it, so you better fucking remember!). So, now we have a randomly selected number in the chosen constant. Let’s say we picked 6. Time to make the swap!

So, remember our emptyIndex? It’s finally coming into play! WOOHOO! board[emptyIndex] = board[chosen]is a pretty simple line. What it is saying is “take the emptyIndex number [3] in the board array, and make it equal to our chosen number [6]”

This is what our board looks like now

0 1 2
6 4 5
6 7 8

    So, what happened? Well, right now our emptyIndex is now 6 and we’ve replaced the previous emptyIndex on our board with tile 6. Except now we don’t have an empty slot anymore because our emptyIndex and chosen tiles are both 6.

Here’s the array as it’s being stored. 0, 1, 2, 6[emptyIndex], 4, 5, 6[chosen], 7, 8 So, how do we get our empty tile back?

    Easy fix. We just need to set chosen (our original 6) to 3. Voila. Swapped numbers. Now our board looks like this

0 1 2
6 4 5
3 7 8

Now this is our array 0, 1, 2, 6[emptyIndex], 4, 5, 3[chosen], 7, 8

To fix everything and get our 3 back into emptyIndex we need to do a little bit of variable shuffling.

The first variable we need to take care of is lastSwapped. Again, this one is to prevent our selector from picking the same time twice in a row. That’s easily handled by the lastSwapped[-1] = emptyIndex[6]

Then we gotta get our 3 back where it belongs in emptyIndex.

emptyIndex[6] = chosen[3]

Voila. We don’t actually care that both emptyIndex and chosen are now 3 as we won’t be looking at them again until the next move.

And then we finish it off with a variable that keeps track of how many moves we’ve made so far with currentShuffle++; . This just iterates that variable by 1. Make sure that at the beginning of our shuffle() function we check that currentShuffle > maxShuffle and we’re done! It’ll go through the shuffle() function 299 more times just to make sure that things are really scrambled. I could have chosen a lower number, something like 50 probably would have been enough but computers are so damn powerful now that even a modern calculator could probably run through that function all 300 times faster than you can blink. You’d probably have to hit the tens of thousands of iterations for it to begin to be noticable.

Here’s the relevant parts of the live code that I talked about tonight. I left out a few minor details here and there in this post but if you were able to follow along then you won’t have any problems deciphering the rest.

const ROWS = 3;
const COLS = 3;
var board = [0, 1, 2, 3, 4, 5, 6, 7, 8];
var emptyIndex = 3;
const solved = [0, 1, 2, 3, 4, 5, 6, 7, 8];


function shuffle() {
    let lastSwap = -1;
    const shuffleMax = 300;
    let currentShuffle = 0;

    while (currentShuffle < shuffleMax) {
        const candidates = [];
        const left = emptyIndex - 1;
        const right = emptyIndex + 1;
        const above = emptyIndex - COLS;
        const below = emptyIndex + COLS;
        // Check left neighbor (index - 1): valid if in same row
        if (left >= 0 && Math.floor(left / COLS) === Math.floor(emptyIndex / COLS)) {
            candidates.push(left);
        }
        // Check right neighbor (index + 1): valid if in same row
        if (right < ROWS * COLS && Math.floor(right / COLS) === Math.floor(emptyIndex / COLS)) {
            candidates.push(right);
        }
        // Check above neighbor (index - COLS): valid if result >= 0
        if (above >= 0) {
            candidates.push(above);
        }
        // Check below neighbor (index + COLS): valid if result < total tiles
        if (below < ROWS * COLS) {
            candidates.push(below);
        }
        const filtered = candidates.filter(c => c !== lastSwap);

        const chosen = filtered[Math.floor(Math.random() * filtered.length)];

        // Swap chosen tile with empty slot in the board array
        board[emptyIndex] = board[chosen];
        board[chosen] = 3; // 3 is the empty tile identity
        lastSwap = emptyIndex;
        emptyIndex = chosen;
        currentShuffle++
    };

};
